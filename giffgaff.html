Copy to Clipboard function

I needed to allow the ability for a developer to copy text, code, or an icon to their clipboard, without them having to highlight what they needed on the design system site. To do this, I added an onclick function on anything that required copying. As the design system site is built using Hugo templating language, this only required a few places for the onclick to be inserted, and it acts across the entire site. I then created a placeholder function, which is required for copying to clipboard. I then created a seperate function for each different thing that needed copying, and targeted those seperately, calling the placeholder function inside each one so as to avoid repeating code.

I included a span element, which appears when the user hovers over or focuses on the item that can be copied. This allows the site to be fully accessible, as well as adding functionality. For the SVG icons, I added small icon buttons, which allow the developer to choose whether they'd like to copy the SVG code or the SVG URL, allowing them to be used in any situation. For the copying code function, I added a button that can be clicked to allow the developer to copy the entire code block to their clipboard. Once clicked, this button's text changes from 'Copy' to 'Copied' to give them a reference that the action has been completed.

FAQ Shortcode

I needed to create a valid Hugo shortcode for a Frequently Asked Questions section for use in the giffgaff design system site, to help other developers keep the giffgaff website looking uniform and on-brand throughout the site. I also has to include a section in the documentation to explain how this should be used. I had a few difficulties whilst completing this, the first one being that I needed the developer to be able to add as many list items to their FAQ section as they needed to.

To solve this, I created two shortcodes, the first one is the ul shortcode, which surrounds the list. Inside the ul shortcode, the developer can then add as many versions of the li shortcode as they'd like, to correspond with how many FAQs they need to include. The li shortcode then takes the content of the li as a variable, which allows the developer to add thier FAQ text and even code, for it to display on the website how they would like.

Design Token Imports

The giffgaff design system is set up from the styleguide. On the design system site, it wasn't automatically updating from the styleguide, which meant that any time the styleguide was changed or added to, the design system site had to be manually updated to match. This was a waste of time at best, and at worst, it could end up with design styles being used incorrectly, or just missed altogether.

My challenge was to ensure that this updating was automated, as well as making sure that the code was displayed in the correct coding language format for each developer depending on how they were using the styles. I set up a language shortcode in Hugo with an if / else statement, which takes the styleguide directly from the source, and takes the language entered as the shortcode variable.

I then used PrismJS to find the correct formatting for each language, and ran the styleguide through the relevant formatting before displaying it on the design system site. My only lingering issue from this is that LESS doesn't display with the correct colour highlighting, which appears to be a small issue with PrismJS. I ran this through as CSS instead to get an easily readable format.

SIM order and success page analysis

This was my first project from within the BI & Ops team. I was asked to take a look at the three free SIM order pages, and look at whether those pages were underperforming in any way, presenting a hypothesis and a potential solution.

I started on Google Analytics, looking at the user journeys to and from each page, and the devices they were accessing these page from. I noticed a strange trend in which many users were exiting immediately from the 'Order a SIM' pages. This was concerning as we don't want people leaving the website before ordering a SIM. Looking further into this, I found that users on the 'Success page' after ordering a SIM card were returning back to the 'Order a SIM' page. Why would a user return to Order a SIM when they've already ordered one? This led me to believe that users weren't being given the continuation options that they wanted after ordering a SIM card.

I found that the navigation bar was removed on the Order a SIM and Success pages, to streamline the user into the SIM ordering process. However, this left them with a lack of future links.

    I had two hypotheses:
  • The users were returning to the 'Order a SIM' page to view the FAQs, which were visible on the 'Order a SIM' page, but not on the 'Success page'. This could be fixed by linking the FAQs on the 'Success page'.
  • The users were pressing the 'Back' button on their browsers, because they didn't want to follow any of the three options that the 'Success page' gives them.

I created 3 variants in Optimize, the first variant with the FAQs also listed on the 'Success page', the second variant adding a button to link to the FAQs on a seperate page, and a third variant with a link to the giffgaff community Hub. I presented these hypotheses and possible solutions to the team, which I then built as an A/B test in Optimize. It is currently waiting to go live.
